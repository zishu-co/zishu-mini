# SDD（规格驱动开发）规范

> Specification-Driven Development，规格驱动开发

## 核心理念

**先规格，后代码** - 所有开发工作从编写规格文档开始，确保需求清晰、目标明确后再动手编码。

---

## 📝 规格文档要求

### 1. 规格文档命名规范

| 文档类型 | 命名格式 | 示例 |
|---------|---------|------|
| 页面规格 | `page-{页面名}-SPEC.md` | `page-goods-SPEC.md` |
| 功能规格 | `feature-{功能名}-SPEC.md` | `feature-login-SPEC.md` |
| 组件规格 | `component-{组件名}-SPEC.md` | `component-modal-SPEC.md` |

### 2. 规格文档模板

```markdown
# {功能/页面名称}

## 1. 概述
- **功能描述**: 一句话描述功能
- **业务背景**: 为什么需要这个功能
- **优先级**: P0/P1/P2/P3

## 2. 页面结构
- **页面路由**: 路由路径
- **页面流程**: 用户操作流程

## 3. 功能需求
### 3.1 功能点列表
| 功能点 | 描述 | 验收标准 |
|-------|------|---------|
| xxx   | xxx  | xxx     |

### 3.2 用户交互流程
1. 用户进入页面
2. 用户执行操作
3. 系统响应

## 4. 数据模型
### 4.1 页面状态
```typescript
interface IPageData {
  // 状态定义
}
```

### 4.2 全局数据更新
- 需要更新 `app.globalData` 的场景

## 5. API 接口
| 接口 | 方法 | 描述 |
|-----|------|-----|
| /api/xxx | POST | xxx接口 |

## 6. 组件使用
| 组件 | 用途 | 来源 |
|-----|------|-----|
| xxx | xxx | 内部组件库 |

## 7. 验收标准
- [ ] 功能点1
- [ ] 功能点2
- [ ] 边界条件处理

## 8. 更新记录
| 日期 | 版本 | 更新内容 | 作者 |
|-----|------|---------|------|
| yyyy-mm-dd | v1.0 | 创建 | xxx |
```

## 🔄 SDD 开发流程

```
┌─────────────┐
│ 1.需求分析   │
│ 获取需求文档  │
└──────┬──────┘
       ▼
┌─────────────┐
│ 2.编写规格   │◄──────────────────┐
│ 编写SPEC.md │                   │
└──────┬──────┘                   │
       ▼                          │
┌─────────────┐                   │
│ 3.评审规格   │                   │
│ 团队评审通过  │                   │
└──────┬──────┘                   │
       ▼                          │
    ┌───────┐    需要修改          │
    │ 通过? │─────────────────────┘
    └───┬───┘
        │ 是
        ▼
┌─────────────┐
│ 4.编码实现   │
│ 按规格开发   │
└──────┬──────┘
       ▼
┌─────────────┐
│ 5.自测验收   │
│ 对照规格自测 │
└──────┬──────┘
       ▼
┌─────────────┐
│ 6.CodeReview │
│ 团队评审     │
└──────┬──────┘
       ▼
┌─────────────┐
│ 7.合并发布   │
│ 合并到主分支 │
└─────────────┘
```

## ✅ 验收检查清单

开发完成后，对照 SPEC.md 检查：

- [ ] 所有功能点是否按规格实现
- [ ] 页面交互是否符合规格描述
- [ ] 数据结构是否与规格一致
- [ ] API 接口是否符合规格定义
- [ ] 边界条件和异常情况是否处理
- [ ] 类型定义是否完整
- [ ] 组件使用是否正确
