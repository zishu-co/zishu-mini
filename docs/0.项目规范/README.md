## 🎯 快速开始

### 新成员入职

1. 阅读 [SDD开发规范](./SDD开发规范.md) 了解开发流程
2. 阅读 [代码风格指南](./代码风格指南.md) 了解编码规范
3. 查看 [产品规格总览](../1.产品规格/SPEC.md) 了解产品功能

### 新功能开发

1. 在 `docs/1.产品规格/` 下创建功能规格文档
2. 提交评审通过后开始开发
3. 开发完成后对照规格文档验收
4. 提交 PR 进行 Code Review

---

## ⚠️ 核心原则

1. **先文档后代码** - 任何功能开发前必须先完成 SPEC.md
2. **类型安全** - 所有 TypeScript 代码必须有类型定义
3. **组件复用** - 优先使用现有组件，减少重复代码
4. **Code Review** - 所有代码必须经过评审才能合并
