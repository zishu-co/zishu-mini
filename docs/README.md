## 📋 SDD 开发流程

所有功能开发必须遵循以下流程：

```
1. 需求分析 → 编写 SPEC.md 规格文档
2. 技术设计 → 更新架构文档和组件清单
3. 编码实现 → 按规格文档开发
4. 自测验收 → 对照规格文档验收
5. Code Review → 团队评审
6. 合并发布 → 合并到主分支并发布
```

## ⚠️ 注意事项

1. **禁止直接修改主分支** - 所有功能开发必须在特性分支进行
2. **先文档后代码** - 任何功能开发前必须先完成 SPEC.md
3. **组件复用优先** - 优先使用现有组件，新组件需评审
4. **类型必须定义** - 所有接口和全局类型必须显式声明
